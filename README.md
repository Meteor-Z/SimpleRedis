## 侵入式数据结构

### 优点

- 非侵入式数据结构中,数据结构存储的数值是用户传递的的副本(也就是会拷贝以下),但是侵入式数据结构存储的不是数据结构的副本,而是存储对象的本身,对象插入容器所需要的数据要由对象本身提供,
  - 根本不会调用任何内存分配相关的问题(所以根本没有析构函数, 也就是内存分配相关的, 不能使用,应该交给外部进行管理)
  - 提供可预见性, 不会进行内存管理, (内存管理是不可预测的操作),  所以管理起来要比非侵入式的提供的保证更加宽松
  - 比非侵入式的数据结构提供了更好的异常保证,(因为没有所谓的内存分配的问题)
- 更快的访问速度，【完全没有内存分配】，内存访问要快很多
- 不会存储对象的copy，所以说有些对象可能不能进行拷贝，那么就可以使用侵入式数据结构

### 缺点

- 存储的是节点,所以要改变原来的对象的属性, 也就是内存布局，最起码要加入两个指针，所以内存分布要变大，还要修改其中类的定义
- 用户要独立于容器管理插入对象的生命周期(因为内存管理是外部的)
